이 버전은 FFmpeg Worker를 사용하지 않으므로 GitHub Pages에서 worker 경로 오류를 피할 수 있습니다.
다만 브라우저가 해당 영상의 오디오를 디코딩할 수 있어야 하며, 긴 영상은 메모리를 많이 사용할 수 있습니다.
권장 배포 환경: GitHub Pages / Netlify / Vercel
사용 라이브러리: lamejs CDN 로드
변환 구조: Video 파일 → Audio 디코드 → PCM → MP3 인코딩
